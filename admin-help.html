Admin Help Guide

Running Your Auction & Tombola

Everything you need to set up, manage, and close your event — from adding items to notifying winners on WhatsApp.

01 Logging In

Open admin.html in your browser. You will see a PIN entry screen. Enter your admin PIN and press Enter or click the button.

Where is my PIN? Your PIN is stored securely in Google Apps Script — not in the spreadsheet. To change it, go to your Apps Script project → ⚙ Project Settings → Script Properties → find AdminPIN and update the value.
PIN not working? Open Apps Script → run the debugPin() function → check the Logs. If it shows {} the PIN was never saved. Go to Project Settings → Script Properties → Add property: key AdminPIN, value your chosen PIN → click Save script properties.
02 Dashboard Overview

After logging in, the top bar shows four navigation sections. Click any to switch between them.

🏺
Items
Add, edit and manage auction items. See stats, close individual items, share to WhatsApp.
📊
Live Bids
See the current highest bid and leader for every open item in real time.
🏆
Winners
After closing the auction, see every winner and notify them directly on WhatsApp.
🎟
Tombola
Manage tombola prizes, sponsor videos, run draws and notify tombola winners.

The auction name and close date shown in the topbar come from your Config sheet. Update AuctionName and AuctionEnd there to change what appears.

03 Managing Auction Items

The Items section is your main workspace before and during the auction.

Adding an item:

1
Click + Add Item

Opens the item form on the right.

2
Fill in the details

Name and Starting Bid are required. Description, image and minimum increment are optional but recommended.

3
Add an image

Drag and drop an image file onto the upload area, or click it to browse. The image uploads to ImgBB automatically. You can also paste a direct image URL instead.

4
Click Save Item

The item appears in the table immediately and is live on the public site.

The stat cards at the top of the Items section always show a live count of total items, how many are open, how many are closed, and the total value of all current bids combined.

Action buttons per item:

ButtonWhat it does
EditOpen the item form pre-filled — change name, description, image or starting bid. Does not reset current bids.
ShareOpens WhatsApp with a pre-written message about this item — current bid, leader, close date and your site URL included.
CloseLocks bidding on this item only. The current highest bidder wins. Only appears on open items.
DelPermanently deletes the item. Shows a confirmation first.
Min Increment The minimum increment is the smallest amount a new bid must exceed the current bid. For example if current bid is R 500 and increment is R 50, the next bid must be at least R 550. Set this per item.
04 Watching Live Bids

The Live Bids tab shows the current highest bid and leading bidder for every item that has received at least one bid. Items are sorted by bid amount, highest first.

This view does not auto-refresh. Click the ↻ Refresh button to pull the latest data from the sheet. During an active auction, refresh every few minutes to stay current.

Tip Keep the Live Bids tab open on a screen visible to guests at your event — it makes a great live leaderboard display. Just keep refreshing it.
05 Closing the Auction & Viewing Winners

When you are ready to end bidding for all items at once, use the Close All Bidding button in the Items toolbar. This:

To close a single item without closing everything, use the Close button on that item's row instead.

The Winners tab lists every item with a winner, their name, phone number and winning bid. Each row has a Notify on WA button that opens WhatsApp with a congratulations message pre-filled — ready to send in one tap.

Important Close All cannot be reversed easily. Make sure bidding time is genuinely over before clicking it. Individual items can be re-opened manually in the Google Sheet if needed.
06 WhatsApp Sharing

There are four places in the admin where you can share to WhatsApp. All open WhatsApp with a message pre-written — you just choose which contact or group to send it to.

WhereWhat it shares
Share Top 3 buttonThe 3 most bid-on open items — names, current bids, leaders, close date and your site link.
Share button per item rowA single item — name, description, current bid, leader, close date and site link.
Winners tab — Notify on WAA congratulations message to that winner — item name, amount and collection instructions.
Tombola — Share per prizeA single tombola prize — name, sponsor, how to enter, close date and site link.
Best practice Share to your community WhatsApp group before the event opens, again halfway through to build excitement, and once more 30 minutes before closing. The Share Top 3 message works well for this.
07 Managing Tombola Prizes

Navigate to Tombola → Prizes. You will see the same layout as the auction items — stat cards at the top, a toolbar, and a table of prizes.

Adding a prize:

1
Click + Add Prize

Opens the prize form.

2
Fill in details

Prize name is required. Sponsor name, description, image and max tickets are optional.

3
Set max tickets

Leave as 0 for unlimited entries. Set a number (e.g. 100) to cap how many tickets can be earned for this prize.

4
Click Save Prize

Prize is live on the public tombola tab immediately.

Action buttons per prize:

ButtonWhat it does
EditChange prize name, description, sponsor or image. Does not affect tickets already earned.
ShareOpens WhatsApp with a message about this prize — name, sponsor, how to enter, close date and site link.
DrawRandomly selects one winning ticket from all entries for this prize. Shows a confirmation first. Only appears on open prizes.
DelPermanently deletes the prize. Shows a confirmation first.
Tombola must be active Participants can only see the Tombola tab if TombolaActive is set to TRUE in your Config sheet. Set it to FALSE before the event to show a "coming soon" message, then switch to TRUE when ready to open.
08 Managing Sponsor Videos

Participants earn tombola tickets by watching 30-second sponsor videos. Each completed view awards one ticket. You manage those videos here under Tombola → Sponsor Videos.

Before adding a video you need a YouTube video ID. Here is how to get one:

1
Get the sponsor's video file

Ask your sponsor for an MP4 file, 30 seconds or less.

2
Upload to YouTube as Unlisted

Go to YouTube Studio → Upload → set visibility to Unlisted (not Private or Public). Unlisted means it won't appear in search but will play on your platform.

3
Copy the video ID

From the YouTube URL youtube.com/watch?v=dQw4w9WgXcQ — the ID is the part after ?v=. Copy only that part, not the full URL.

4
Click + Add Sponsor Video

Enter the sponsor name and paste the YouTube ID. Click Add Video.

Video table columns:

ColumnMeaning
SponsorThe business name you entered.
YouTube IDThe video identifier — use the Preview on YT link to verify the right video loaded.
ViewsHow many times this video has been served to participants (counted when the token is issued, before completion).
StatusActive videos are served to participants. Inactive ones are skipped.
Multiple sponsors Add as many sponsor videos as you like. When a participant clicks to earn a ticket, one video is picked at random from all active videos. With three sponsors each gets roughly equal exposure across all ticket earns.
No videos = no tickets If there are no active sponsor videos, participants will see an error when trying to earn tickets. Always have at least one active video before setting TombolaActive to TRUE.
09 Drawing Tombola Winners

You have two options — draw prizes one at a time, or draw all at once.

Draw one prize: Click the Draw button on a prize row. A confirmation appears showing the prize name. Confirm and the system picks a random winning ticket from all valid entries. The result shows immediately — winner name and winning ticket number.

Draw all prizes at once: Click the Draw All Prizes button in the toolbar. All open prizes are drawn simultaneously. After confirming, you are taken to the Tombola Winners tab automatically.

Cannot be undone Once a prize is drawn its status changes to Drawn and the Draw button disappears. The winner is recorded permanently. If you need to re-draw, you would need to manually update the TombolaWinners and TombolaItems sheets in Google Sheets directly.
10 Tombola Winners

After drawing, go to Tombola → Winners. Each prize winner is listed with their name, phone number, winning ticket ID and the time the draw was made.

Click Notify on WA next to any winner to open WhatsApp with a pre-written congratulations message that includes the prize name, their winning ticket number and a request to arrange collection.

No phone number? If a winner registered without a phone number the Notify on WA button will not appear. Contact them through whatever information you have from registration.
11 Config Sheet Reference

Open your Google Sheet and click the Config tab. The Key column is the setting name — never change it. The Value column is what you edit.

KeyWhat to put here
AuctionNameYour event name — shown in the public page header and all WA messages.
AuctionEndDate and time bidding closes. Format exactly: 2025-11-30T20:00:00
WhatsAppNumYour WhatsApp number, digits only, with country code. e.g. 27831234567 not 083 123 4567
SiteURLThe full address of your public page. e.g. https://silentauction.morneydeetlefs.workers.dev/
TombolaNameYour tombola display name. e.g. Harvest Tombola 2025
TombolaEndDate and time tombola closes. Same format: 2025-11-30T21:00:00
TombolaActiveTRUE to show tombola to participants, FALSE to hide it.
TicketsPerViewHow many tickets each completed video earns. Usually 1.
TokenExpiryHow many seconds after a video starts the ticket claim is valid. Leave as 45.
After changing the Config sheet Changes take effect immediately — no redeployment needed. The public page and admin both fetch config live on each load. Refresh the page to see changes.
12 Event Day Checklist

Before opening to participants:

  • Config sheet filled in — AuctionName, AuctionEnd, WhatsAppNum, SiteURL
  • All auction items added with images, starting bids and increments
  • TombolaActive set to TRUE in Config
  • At least one sponsor video added and set to Active
  • All tombola prizes added with images and sponsor names
  • Test registration on the public page on your own phone
  • Test placing a bid on one item
  • Test watching the sponsor video and earning a ticket

During the event:

  • Share top 3 auction items to your WhatsApp group to build excitement
  • Share tombola prizes to encourage participation
  • Refresh Live Bids tab regularly to monitor activity
  • Close individual items as their time expires if running staggered closes

Closing the event:

  • Announce on WhatsApp that bidding closes in 10 minutes
  • Click Close All Bidding — confirm when ready
  • Go to Winners tab — notify each auction winner via WhatsApp
  • Go to Tombola → Draw All Prizes — confirm
  • Go to Tombola → Winners — notify each tombola winner via WhatsApp
13 Troubleshooting
ProblemSolution
Incorrect PIN errorGo to Apps Script → Project Settings → Script Properties. Check AdminPIN exists and has no spaces. Run debugPin() and check Logs to confirm the value.
Items not loadingCheck the API constant in both HTML files matches your current deployed Web App URL. Every new deployment creates a new URL — update both files.
Error loading winnersThe Winners sheet may not exist. Run setupSheets() in Apps Script to create it. Also check that Close All has been run — winners only exist after closing.
Tombola tab not visibleSet TombolaActive to TRUE (uppercase) in the Config sheet. Refresh the public page.
No active sponsor videos errorGo to Tombola → Sponsor Videos. Ensure at least one video is set to Active. If the table is empty, add a video first.
Video not completed error on ticket claimThe video must play for at least 28 seconds before the ticket claim is accepted. This is intentional — the full ad must be watched.
Token expired errorThe participant waited too long after the video ended before the ticket was claimed. They need to watch a new video. Increase TokenExpiry in Config if this happens often.
Code changes not workingYou must redeploy after every change to Code.gs. Go to Deploy → Manage deployments → Edit → change Version to New version → Deploy.
Winners list shows dates as numbersDates in your sheet are stored as serial numbers. This is a Google Sheets formatting issue. The platform handles this automatically — if dates look wrong in the admin, check the sheet cell is formatted as Plain Text.
WhatsApp link opens but message is garbledMake sure no emoji are in your Config values — AuctionName, TombolaName etc. Emoji in URL-encoded WA messages can display incorrectly on some devices.
